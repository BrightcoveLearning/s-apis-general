Manifest tags

The ingest manifest is using to add video metadata to a video. The manifest may contain both Brightcove tags and Ooyala tags (for compatibility with the old Ooyala ingest manifests).

Brightcove tags

The tags detailed below must be enclosed in a <brightcove></brightcove> tag.

Notes:

  • All special characters must be escaped.
  • Labels and Tags must be separated by a comma.

You can find additional detail on the metadata fields in the CMS API Video Fields Reference Note that the overall manifest should be constructed as an RSS feed, with the Video Cloud metadata includes as channel items. In other words, the overall manifest should be structured like this:

<rss xmlns:media="http://search.yahoo.com/mrss/" 
  xmlns:dcterms="http://purl.org/dc/terms/" 
  xmlns:fh="http://purl.org/syndication/history/1.0" 
  xmlns:brightcove="http://www.brightcove.com/mrss/" version="2.0">
  <channel>
    <item>

    <!-- items included here are constructed using the tags detailed below -->

    </item>
  </channel>
</rss>

Brightcove tag reference

<profile> id/name </profile>
The name or id of the ingest profile to use. Applicable only to videos. If omitted, the default profile for the account will be used.
<name></name>
The title for the video.
<master url=”string” useMaster=”boolean”/>
For a retranscoding request. You can either indicate that a stored master should be used (useMaster="true"), or provide a URL for a master that was not stored. If you include both url and useMaster, the request will fail.
<description long=”string” short=”string”/>
A short description for the video.
<images> array of <image>
Contains a collection of images associated with the video - see the following tag.
<image url=”string/required” language=”string” variant=”string/required” label=”string” height=”number” width=”number”/>
Details for an image (see Dynamic Ingest Fields Reference for more detail). Briefly:
  • language is a two-letter language code from the Language Subtag Registry.
  • variant is poster | thumbnail | portrait | square | wide | ultra-wide.
  • height and width are pixel values.
<texttracks> array of <texttrack>
Contains a collection of text-tracks associated with the video - see the following tag.
<texttrack url=”string/required” srclang=”string/required” kind=”string” label=”string” default=”boolean”/>
See the Dynamic Ingest Fields Reference for details. Briefly:
  • srclang - ISO 639 2-letter (alpha-2) language code for the text tracks
  • kind - captions | subtitles | chapters | metadata | transcriptions
  • label - user-readable label
  • default - whether these text-tracks are the default for the video
<audiotracks> array of <audiotrack>
Contains a collection of audio tracks associated with the video - see the following tag.
<audiotrack url=”string” language=”string” variant=”string”/>
See the Dynamic Ingest Fields Reference for details. Briefly:
  • language - ISO 639-2 3-letter language code
  • variant - main | alternate | commentary | dub | descriptive
<customField name="string/required">value</customField>
The value will be assigned to the custom field with the given name for this video.
<labels> separated by comma </labels>
See Working with Labels.
<tags> separated by comma </tags>
A list of tags. Tags can be used to select videos for smart playlists, among other things.
<referenceId></referenceId>
A unique reference id for the video. Reference ids can be used to associate the video with an entity in your systems, and you can also publish videos using the reference id rather than the video id.
<videoId></videoId>
The video id. Do not include this if you ingesting a new video. To be used only for replacing or retranscoding or adding media assets to an existing video.
<remoteasset url=”string/required” name=”string” durationInMs=”number” encodingRate=”number” frameHeight=”number” frameWidth=”number” size=”number” videoCodec=”string” videoContainer=”number” progressiveDownload=”boolean”/>
For adding a remote asset video. Note that if you include both the <master> and <remoteasset tags, the request will fail.
<schedule endsAt=”” startsAt=””/>
For scheduling when a video becomes available for playback, and (optionally) when it becomes unavailable again. Times in ISO-8601 date format such as 2022-09-20T20:41:07.689Z or just 2022-09-20.
<link> value </link>
URL for a related link.

Sample manifest (Brightcove schema)

Below is a sample manifest for a video ingest. Note that if you are only using Brightcove tags, you should the xmlns:brightcove attribute (highlighted - compare the later sample that uses Ooyala tags).

<rss xmlns:media="http://search.yahoo.com/mrss/" 
    xmlns:dcterms="http://purl.org/dc/terms/" 
    xmlns:fh="http://purl.org/syndication/history/1.0" 
    xmlns:brightcove="http://www.brightcove.com/mrss/" 
    version="2.0">
    <channel>
      <item>
  
        <brightcove>
          <master url=""/>
           <profile>multi-platform-standard-dynamic-with-mp4</profile>
           <name>Title OIS V1-MRSS-04042016_00008 MC</name>
           <status>ACTIVE</status>
           <description short="desc" long="Desc OIS V1-MRSS-04042016_00008 MC" />
           <labels>/nihanth1/dara1,/harsha1/kanna1</labels>
           <tags>birds,sea</tags>
           <referenceId>asdf1234</referenceId>
           <videoId>73446256700bol1</videoId>
           <customField name=”season_number”>1</customField>
           <remoteasset url="//learning-services-media.brightcove.com/videos/mp4/greatblueheron.mp4
  "></remoteasset>
           <schedule startsAt="2021-05-01T18:00:00.000+08" endsAt="2021-05-08T18:00:00.000+08"/>
           <link>http://mysite.com/videos/sports/real_sports.html</link>
           <images>
             <image url="http://ots-testing.ooyala.com.s3.amazonaws.com/MRSS_103469/Assets_03012016/images/big_buck_bunny_1.png" variant="thumbnail" language="es" label="thumbnail" height="480" width="640"/>
           </images>
           <texttracks>
             <texttrack url="http://ots-testing.ooyala.com.s3.amazonaws.com/MRSS_103469/Assets_03012016/SubTitles/cc_030152016-2.vtt" srclang="en-US" kind="captions" label="English"/>

    </channel>
  </rss>

Ooyala tags

The tags detailed below are compatible with the old Ooyala ingest manifest and are intended for former Ooyala customers to enable them to use existing manifests.

Notes:

  • Retro-compatible with Ooyala XML (we use the Ooyala schema if it’s a Ooyala header)
  • Ooyala MRSS supports only one Subtitle
  • Ooyala MRSS does not support audio tracks
  • Brightcove MRSS will support multiple subtitles
  • Brightcove MRSS will support multiple audio tracks
  • Brightcove MRSS: all the tags required will be inside a parent tag <brightcove></brightcove>

Ooyala tag reference

Note that Brightcove tags can also be included with Ooyala tags but must be enclosed in a <brightcove></brightcove> block.
<ooyala:profileguid>
Specifies a Video Cloud ingest profile to use for processing the video. If you do not want to use the account default profile, you must specify the one you want to use instead. See Managing Ingest Profiles.
<media:title></code>
The video title.
<media:content url=””>
URL is for the video file to be ingested
<media:description>
Short description for the video
<media:thumbnail url=””>
URL for video thumbnail image
<media:subTitle href=””>
The href value is the URL for a captions file for the video
<ooyala:labels> separated by commas </ooyala:labels>
Labels for video, comma-separated - you can also use the Brightcove <labels></labels> tag
<media:keywords> separated by commas </media:keywords>
Video tags, separated by commas - you can also use the Brightcove <tags></tags>
<ooyala:embedcode>video id</ooyala:embedcode>
The Video Cloud video id - required if you are updating an existing video
<ooyala:metadata name=”key”> value </ooyala:metadata>
Set a value for the custom field identified by the key
<ooyala:remoteasset durationInMs=”” url=””>
For creating remote asset videos - if remoteasset and media:content are both provided, the request will fail
<dcterms:valid>
Flight times representing the start and end times when the asset can be played. Example: 
<dcterms:valid>
  start=2011-06-28T15:15:00-07:00;
  end=2022-12-01T16:00:00-07:00;scheme=W3C-DTF
  </dcterms:valid>
<>
Creates a related link - in the old Ooyala system, this field was used to store the URL where the video was embedded and can still be used for that purpose if you wish
<ooyala:metadata name="video_ID">71</ooyala:metadata>
Can be used instead of <ooyala:embedcode> to identify an existing video you are adding/updating metadata for
<ooyala:metadata name="reference_ID">71</ooyala:metadata>
Can be used instead of <ooyala:embedcode> to identify an existing video you are adding/updating metadata for
